# 📘 商品管理操作手冊

這份手冊說明你之後要新增、修改、下架商品時該怎麼做。

> 💡 **核心觀念**：你只需要修改 **Excel 檔案**（你原本就在用的那個），然後執行「一鍵同步」就會自動把所有變更套用到網站。**不需要碰程式碼**。

> 📌 **關於這個團購活動**：本次 PIF 難民特賣會是**單一檔次活動**，所有訂單都會進到同一份 Google Sheets。你之後只需要持續修改現有的 Excel 檔（增刪商品、改價格、換圖片等），不需要建立新的活動。

---

## 🛠️ 第一次使用前準備

只需要做這一次：

### 1. 安裝 Python（如果電腦裡還沒有）

1. 打開 https://www.python.org/downloads/
2. 點擊黃色大按鈕「Download Python」下載
3. 安裝時 **務必勾選「Add Python to PATH」** 這個選項（很重要！）
4. 一直按下一步完成安裝

### 2. 安裝必要套件

按下 `Win + R` → 輸入 `cmd` → 按 Enter，在黑色視窗輸入：

```
pip install openpyxl
```

按 Enter 等它跑完即可。

### 3. 測試一鍵同步

雙擊 `一鍵同步.bat`，看到「✅ 同步完成」就表示準備好了。

## 📝 三大常用情境

### 情境 A：修改一兩個商品的價格 / 名稱（最常見）

1. 打開 Excel `【第一波】PIF難民特賣會.xlsx`
2. 找到要改的商品那一列，直接修改：
   - **單價**（E 欄）
   - **品名**（D 欄）
   - **每箱入數**（F 欄）
3. 存檔（Ctrl + S）
4. **關閉 Excel**（很重要！同步腳本無法讀取已開啟的檔案）
5. 雙擊 `一鍵同步.bat`
6. 重新整理瀏覽器（按 `Ctrl + F5` 強制重整）看效果

⏱️ **耗時**：約 30 秒

---

### 情境 B：批次新增商品（例如進新貨）

#### Step 1：在 Excel 加入新商品列

1. 打開 Excel
2. 在商品表格的最後一列下面，依照原本的格式新增商品列：

   | No. | 商品圖 | 條碼 | 品名 | 單價 | 每箱入數 | 訂購數量 | 小計 |
   |-----|-------|------|------|-----|--------|--------|------|
   | 84  | （貼圖）| 1234567890 | 【XX】商品名稱 | 100 | 12 | | `=IF(G96="","",E96*G96)` |

3. **重點**：
   - **No. 要連續編號**（接續上一筆，不要跳號）
   - **小計欄的公式**要依新行號修改（例如新行號是 96，公式就是 `=IF(G96="","",E96*G96)`）

#### Step 2：在「商品圖」欄貼上圖片

1. 從手機/電腦複製商品圖片
2. 點選 B 欄該列的儲存格
3. 上方選單 **插入 → 圖片 → 此裝置**
4. 選擇圖片，調整尺寸讓它剛好放進儲存格

#### Step 3：執行同步

存檔 → 關閉 Excel → 雙擊 `一鍵同步.bat`

⏱️ **耗時**：每個新商品約 1 分鐘

---

### 情境 C：完全下架某個商品

> 💡 想暫時不出貨某商品時，使用這個方法。

#### 方法 1：直接從 Excel 刪除整列（推薦）

1. 在 Excel 中，**右鍵點該商品的列號** → **刪除**
2. **重要**：刪除後要**重新編號** No. 欄（讓編號連續）
   - 例如刪除原本 No.5，那 No.6~83 要全部往前一號
   - 方法：在 A 欄第一個商品輸入 1，第二個商品輸入 2，選取兩格 → 拖右下角的小方塊往下拉到最後一筆，會自動填入連續編號
3. 存檔 → 關閉 → 一鍵同步

#### 方法 2：用「庫存設為 0」隱藏（不破壞編號）

如果你不想動編號，可以在品名後面加上「⚠ 僅剩 0 個」：

```
【XXX】商品名稱
⚠ 僅剩 0 個
```

這樣網站上會顯示「僅剩 0 個」，客人無法加購。但商品還是會出現在列表中。

> 適合：要暫時下架但之後可能會回補貨的商品。

## 💡 常用技巧

### 技巧 1：標示「即將售完」的商品

只要在 Excel 商品名稱的第二行加上 `⚠ 僅剩 X 個`，網站右上角就會自動顯示橘色庫存警示徽章，且購買數量上限會自動限制：

```
【XXX】商品名稱
⚠ 僅剩 5 個
```

### 技巧 2：商品名稱換行顯示「規格」

在 Excel 品名欄按 `Alt + Enter` 換行，第二行會自動變成網站上的灰色規格說明：

```
【XXX】商品名稱
460mL+460mL
```

第一行是商品主名稱（粗體大字），第二行是規格（小字灰色）。

### 技巧 3：讓某個商品「特別推薦」或「置頂」

兩種方法：
- **改 No. 編號**：把它改成 1，網站列表會排在第一個（其他商品要往後排）
- **在品名加 emoji**：例如 `🔥【XXX】商品名稱`，視覺上會比較醒目

### 技巧 4：修改網站公告區塊（庫存清倉通知）

這個區塊的文字是寫在 `index.html` 裡，直接告訴 Claude「請幫我把庫存清倉通知的文字改成 OO」就能更新。

## ❓ 常見問題

### Q1：執行「一鍵同步.bat」沒反應 / 視窗一閃就關閉？

可能原因：
- **Python 沒裝好或沒勾 PATH**：重裝一次，記得勾「Add Python to PATH」
- **Excel 還開著**：請先關閉 Excel 再執行
- **缺套件**：在 cmd 跑 `pip install openpyxl`

### Q2：訂購須知（取貨地點、運費說明等）改了 Excel，網站沒更新？

是的，目前的訂購須知和庫存清倉公告都是直接寫在 `index.html` 裡。

如果要改取貨地點、匯款資訊或公告內容，有兩種方式：
1. **直接編輯 `index.html`**：用記事本打開，找到對應段落（例如 `<section class="info-section">` 或 `<section class="clearance-banner">`）自己改文字
2. **告訴 Claude 幫你同步**：說「請依最新 Excel 同步訂購須知到網站」或「請把庫存清倉通知改成 OO」

### Q3：商品名稱裡有特殊符號（如雙引號）會壞掉嗎？

不會。同步腳本會自動跳脫特殊字元。但建議避免在品名中使用 `"` 雙引號和 `\` 反斜線。

### Q4：同步後商品圖片沒換？

瀏覽器有快取。請按 `Ctrl + F5` **強制重新整理**。

### Q5：客人下單會進到哪裡？

所有訂單會自動寫入你部署時建立的那份 Google Sheets，包含兩個工作表：
- **訂單總覽**：每筆訂單一列（含未稅、稅金、應付總額、購買摘要等）
- **訂單明細**：每筆訂單的每樣商品一列（方便揀貨、統計）

你只需要每天打開那份 Google Sheets 看新訂單即可。

### Q6：Excel 檔案改名了，腳本還能找到嗎？

可以。腳本會自動偵測資料夾中的 `.xlsx` 檔。如果有多個 `.xlsx`，會跳出選單讓你選。

### Q7：網站線上版本何時會更新？

部署到 GitHub Pages / Netlify 後，每次你執行同步並把 `website/` 資料夾**重新上傳**到平台，網站才會更新。

- **GitHub Pages**：在 GitHub 網頁上把更新後的檔案上傳，等 1~2 分鐘自動發布
- **Netlify Drop**：拖拉新的 `website/` 資料夾到 https://app.netlify.com/drop ，會立刻更新

> 💡 部署完成後第一次打開的網址記得加到瀏覽器書籤，方便日後使用！

### Q8：可以一次大量新增很多商品（例如 50 個）嗎？

可以。直接在 Excel 一次貼上多筆商品列、貼好圖片，然後同步即可。同步腳本沒有商品數量限制。

## 📂 檔案說明（給好奇心強的你）

```
website/
├─ index.html              ← 網站主檔（介面、樣式、邏輯）
├─ products.js             ← 商品資料（不要手動改，會被同步腳本覆蓋）
├─ info.json               ← 訂購須知資料（同步時自動更新）
├─ images/                 ← 商品圖片
├─ GoogleAppsScript.gs     ← Google Sheets 串接的程式碼
├─ sync_from_excel.py      ← 同步腳本本體（Python）
├─ 一鍵同步.bat            ← 雙擊執行同步腳本
├─ 部署說明.md             ← 第一次架站時的指引
└─ 商品管理手冊.md         ← 你正在看的這份
```
