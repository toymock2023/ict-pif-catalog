# PIF 難民特賣會 - 部署說明

這是一個單頁式團購訂單網站，搭配 Google Apps Script 將訂單資料自動寫入 Google Sheets。

## 📁 檔案結構

```
website/
├── index.html              ← 網站主檔
├── products.js             ← 83 項商品資料
├── images/                 ← 商品圖片（83 張）
├── GoogleAppsScript.gs     ← Google Apps Script 程式碼
└── 部署說明.md             ← 本說明文件
```

---

## 🚀 部署步驟

### 步驟 1：建立 Google Sheets 試算表

1. 前往 https://sheets.google.com/ 建立一個新試算表
2. 將試算表命名為「PIF 難民特賣會 - 訂單」
3. 複製試算表 URL 中的 ID（中間那串字）：
   ```
   https://docs.google.com/spreadsheets/d/【這串就是 ID】/edit
   ```

### 步驟 2：建立 Google Apps Script Web App

1. 在剛建立的試算表中，點擊上方選單：**擴充功能 → Apps Script**
2. 把預設的 `Code.gs` 內容全部刪除
3. 開啟資料夾中的 `GoogleAppsScript.gs`，把全部內容複製貼上
4. **重要**：把第 10 行的 `SPREADSHEET_ID` 改成步驟 1 取得的試算表 ID
   ```javascript
   const SPREADSHEET_ID = "1AbCdEf...你的ID...XyZ";
   ```
5. **(選擇性)** 如果要接收訂單 email 通知，找到 `sendOrderNotification` 函式，把 `recipient` 改成你的 email
6. 點擊上方💾 儲存
7. 點擊右上角的「**部署**」→「**新增部署作業**」
8. 設定：
   - 類型：選擇「**網頁應用程式**」
   - 說明：例如「PIF 訂單接收」
   - 執行身分：**我**
   - 誰可以存取：**任何人**（這樣網站才能不需登入也能寫入）
9. 點擊「部署」
10. 第一次部署會要求授權，請允許 Google 帳號權限
11. 部署完成後會得到一段「網頁應用程式 URL」，**請複製起來**

### 步驟 3：將 Web App 網址設定到網站

1. 開啟 `index.html`
2. 找到這一行（大約在 `<script>` 區塊的開頭）：
   ```javascript
   const GAS_WEB_APP_URL = "請替換為你的 Google Apps Script Web App 網址";
   ```
3. 把網址替換成步驟 2 取得的 Web App URL，例如：
   ```javascript
   const GAS_WEB_APP_URL = "https://script.google.com/macros/s/AKfycbx.../exec";
   ```
4. 儲存檔案

### 步驟 4：上線網站

你有 3 種免費方式可以讓網站上線：

#### 方式 A：GitHub Pages（推薦）

1. 註冊 GitHub 帳號
2. 建立新的 repository（例如 `pif-order`）
3. 把 `website/` 資料夾的所有檔案上傳到 repository
4. 進入 repository **Settings → Pages**
5. Source 選 `main` branch、`/(root)` → 儲存
6. 等待 1～2 分鐘，網站就會有公開網址：
   ```
   https://你的帳號.github.io/pif-order/
   ```

#### 方式 B：Netlify（拖拉就好，最簡單）

1. 前往 https://app.netlify.com/drop
2. 把整個 `website/` 資料夾拖進去
3. 立刻就有網址了！

#### 方式 C：本地測試

直接雙擊 `index.html` 就能在瀏覽器開啟，但這樣沒辦法把網站分享給別人。如果只是要本機展示，這樣就夠用了。

## ✅ 測試訂單流程

部署完成後，建議先做一筆測試訂單：

1. 開啟網站
2. 加幾項商品到購物車
3. 填寫訂購資訊送出
4. 開啟 Google Sheets，應該會看到：
   - **訂單總覽** 工作表：一筆完整的訂單摘要
   - **訂單明細** 工作表：每樣商品都有獨立一列
5. 如果你有設定 email 通知，信箱也會收到一封訂單通知

## 🔧 常見問題

### Q1：訂單沒有寫入 Google Sheets？
- 確認 `index.html` 的 `GAS_WEB_APP_URL` 已替換為實際網址
- 確認 Apps Script 部署時「誰可以存取」選了「任何人」
- 開啟瀏覽器 F12 → Console 查看是否有錯誤訊息

### Q2：每次修改 Apps Script 後都要重新部署嗎？
- 是的。修改程式碼後請點「部署 → 管理部署作業 → ✏️編輯 → 版本選新版本 → 部署」

### Q3：可以新增/修改商品嗎？
- 可以。直接編輯 `products.js`，依照原有格式新增或修改商品即可。
- 圖片放進 `images/` 資料夾，並在 `products.js` 中指定對應檔名

### Q4：想要關閉訂購（截單後）？
- 可以在 `index.html` 的 `<form>` 上方加一行公告，然後把送出按鈕的 `<button>` 改為 `disabled`
- 或在 Apps Script 中加入截止日判斷，超過日期就回傳錯誤

### Q5：圖片載入很慢？
- 第一次載入較慢是正常的（83 張圖片）。`loading="lazy"` 已開啟，會自動延遲載入未捲動到的圖片
- 進階優化可以將圖片壓縮成 WebP 格式

## 📝 程式特點

- ✅ 響應式設計（手機/平板/桌機都能用）
- ✅ 即時搜尋（商品名稱、品牌、條碼）
- ✅ 購物車即時計算總金額
- ✅ 表單驗證（必填欄位、宅配自動顯示地址欄）
- ✅ 訂單編號自動產生
- ✅ 雙工作表設計（總覽 + 明細，方便對帳）
- ✅ 可選 email 通知

## 📞 需要協助？

- 在 Apps Script 編輯器中查看「執行 → 查看執行記錄」
