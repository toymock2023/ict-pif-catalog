# 📦 庫存調整說明（目錄版）

這份說明專門教你：**怎麼改商品庫存，並讓 GitHub 上的網站更新。**

---

## 🔑 先搞懂：庫存怎麼顯示

庫存資料來自 Excel 的 **I 欄（庫存數量）**，同步後會變成網站上的標籤：

| Excel I 欄填的數字 | 網站顯示 | 說明 |
|---|---|---|
| `0` | 灰色「**已售完**」標籤 | 預設會被「隱藏無庫存」開關藏起來 |
| `1` ～ `20` | 橘色「**⚠ 僅剩 X 個**」標籤 | 提醒客人快沒了 |
| `大於 20` 或 **留空** | 不顯示任何標籤 | 代表庫存充足 |

> 規則寫死在 `sync_from_excel.py`，門檻是 20。目錄版和原訂購版共用同一份 `products.js`，所以一次改、兩邊都更新。

---

## ✏️ 調整庫存步驟

### 步驟 1：改 Excel
1. 打開資料夾上層的 **`PIF難民特賣會.xlsx`**
2. 找到要改的商品那一列，修改 **I 欄（庫存數量）**
   - 要標「已售完」→ 填 `0`
   - 要標「僅剩 X 個」→ 填 1～20 的數字
   - 要拿掉庫存標籤（恢復正常）→ 填一個大於 20 的數字，或清空
3. **存檔**，並**完全關閉 Excel**（很重要，沒關會同步失敗）

### 步驟 2：一鍵同步
1. 回到 `website` 資料夾
2. 雙擊 **`一鍵同步.bat`**
3. 視窗會跑出同步進度，看到「**[完成] 同步成功！**」就 OK
   - ⚠️ 若提示找不到 Python，先到 https://www.python.org/downloads/ 安裝，安裝時記得勾「Add Python to PATH」

> 同步做了什麼：讀 Excel → 重新產生 `products.js`（含最新庫存）。

### 步驟 3：上傳到 GitHub（網站才會變！）

**GitHub Pages 不會自動更新**，必須把新的 `products.js` 傳上去：

1. 打開你的 GitHub repository 網頁
2. 點進 **`products.js`** 這個檔案
3. 點右上角的 **垃圾桶旁邊的鉛筆圖示**？不是——改用更簡單的覆蓋上傳：
   - 回到 repo 首頁 → 點 **Add file → Upload files**
   - 把 `website` 裡**新的 `products.js`** 拖進去（同名會自動覆蓋）
4. 下方點綠色 **Commit changes**
5. 等 **1～2 分鐘**，重新整理你的網站網址，庫存就更新了

> 💡 如果同步後圖片也有變動（換了商品圖），記得連 `images` 資料夾一起重新上傳。只改庫存的話，通常只需要傳 `products.js`。

---

## 🧪 上傳前先本機確認（建議）

上傳到 GitHub 前，可以先在自己電腦雙擊 **`index.html`** 打開看看，
確認庫存標籤正確、商品數量沒問題，再傳上去比較保險。

---

## ❓ 常見狀況

**Q：改完 Excel 跑同步，網站還是舊的？**
→ 你少了步驟 3。同步只改了電腦裡的檔案，一定要再上傳 `products.js` 到 GitHub。

**Q：跑一鍵同步出現「偵測到 Excel 還開啟中」？**
→ Excel 沒關乾淨。把 Excel 完全關閉，再雙擊一次 `.bat`。

**Q：我直接在 GitHub 網頁改 `products.js` 的 stockLeft 可以嗎？**
→ 可以，適合臨時改一兩項。但**下次跑一鍵同步時 Excel 會覆蓋掉**你手改的，所以正式調整還是以改 Excel 為準。

**Q：同步後想確認某商品庫存對不對？**
→ 用記事本打開 `products.js`，搜尋商品名稱，看那一行的 `stockLeft` 數字。
   - 有 `stockLeft: 0` = 已售完
   - 有 `stockLeft: 5` = 僅剩 5 個
   - 沒有 `stockLeft` 欄位 = 庫存充足、不顯示標籤

---

整個流程一句話記住：**改 Excel → 雙擊一鍵同步 → 上傳 products.js 到 GitHub**。
